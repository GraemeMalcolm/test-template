This repo is a template for new GitHub repo's containing skilling content in the form of markdown files with embedded media (images, etc.). Additionally, the markdown may contain HTML tags to render video from external sources.

In addition to being the repo template, this repo contains the necessary files to render the content as GitHub pages using Jekyll. In the GitHub Pages site, a main landing page will contain and H1 Heading containing the "title" metadata, a normal paragraph containing the "level" metadata data value, and a paragraph containing the "description" metadata from the readme file, followed by a list of H3 headers with links to the individual pages in the /content folder; using the  "title" metadata value in the content markdown files for the header text, with a small font paragraph showing the "duration" metadata value and a second paragraph showing the "description" metadata value.

Each content page must be rendered with an H1 header based on the title in the readme, and an H2 header based on the title in the content markdown file. The H1 header should be a link to the main site page. The page should include a scrollable area that leaves a narrow margin at both aides and a margin at the bottom that is tall enough to accommodate a "back arrow" link to return to the previous content page (disabled on the first content page) and a "forward arrow" link to open the next content page (disabled on the last content page).
